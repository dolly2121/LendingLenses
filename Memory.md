- 2026-07-16: Root-caused and fixed the Power BI Parquet-read failure ("Parquet magic bytes not found in footer") - superseded the earlier CSV-export workaround (`pipeline/export_gold_csv.py`, deleted). Diagnosis: pyarrow 11.0.0 (this environment's ambient version) failed with `Unrecognized type:24` even reading raw metadata; the on-disk Parquet schema itself was completely standard (verified with `duckdb`'s `parquet_metadata()`/`parquet_schema()` - plain BYTE_ARRAY/UTF8, PLAIN/RLE/RLE_DICTIONARY encodings, nothing exotic). The actual culprit: `polars.DataFrame.write_delta()` always exports via Arrow's *newest* compat level internally (no override available), which encodes string columns as `Utf8View` (a physical Arrow type added in 2023, well after most "stable" Parquet reader vintages including, evidently, Power BI Desktop's bundled reader) into the file's embedded `ARROW:schema` key-value metadata - confirmed by decoding that metadata directly with pyarrow's IPC reader. Fix in `pipeline/lake_io.py`'s `append()`: bypass `write_delta()` and call `to_arrow(compat_level=pl.CompatLevel.oldest())` + `deltalake.write_deltalake()` directly, which writes `large_string` instead - old pyarrow (11.0.0) then reads the file with zero errors. Verified on the real `gold_loan_summary` file (not just a spike): downgraded pyarrow to 11.0.0, confirmed read failure reproduced pre-fix and resolved post-fix, re-upgraded to 25.0.0 afterward. No Gold data changed - same 494 loans, same total_amount to the cent, only the physical on-disk encoding differs. This is now a genuine transitive runtime dependency (not just a diagnostic tool): `write_deltalake`'s Arrow C Data Interface needs a reasonably modern pyarrow to accept the converted table at all (a stale ambient pyarrow breaks writes with `Expected object with __arrow_c_array__ or __arrow_c_stream__ method`) - pinned `pyarrow==25.0.0` in requirements.txt, previously undeclared. Also installed `duckdb` standalone for this diagnosis only (not added to requirements.txt, not referenced by any project code).
