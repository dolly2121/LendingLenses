- 2026-07-15: Pandera-polars edge case found while testing (not present in the real 500-row dataset, only in artificial tiny test batches): when *every* row of a column fails a `not_nullable` check, `failure_cases["index"]` comes back `None` instead of a row number, and the reported check name can also differ (observed `"dtype('String')"` instead of `"not_nullable"`). `silver.py`'s `_validate_loans` now falls back to re-scanning for nulls when index is `None`, and uses `dict.get`/`fail_counts.get(...,0)` instead of direct key lookups so an unrecognized check name is quarantined with a generic reason instead of crashing the run (Rules R6, "never crash the run"). This path cannot fire on the real Bronze batch (always 500 rows, defects are always a small fraction), so no dedicated test was kept for it — see [[phase-3-silver]] if this needs revisiting when the dataset shape ever changes.
