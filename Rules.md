# Rules - boundaries for the AI during the build

**Version 2.0 | These rules override any instinct to add more**

---

## R0. Golden rule

This is a live demo for a senior technical audience, not a product. Every extra file, dependency, or abstraction is a new way to fail on stage. When two options work, take the simpler one. Ponytail's ladder applies to every task: does this need to exist, does the standard library solve it, does an existing dependency solve it, can one line do it.

## R1. Document scale, build simple

Production concerns (scale, security hardening, streaming, retries, auth, monitoring stacks) are handled as **written sections in ARCHITECTURE_MAPPING.md, never as code**, unless a phase explicitly says otherwise. If a production idea comes up mid-build, write it into the mapping doc as one paragraph and move on.

## R2. Libraries

- ALLOWED: polars, deltalake, pandera, fastapi, uvicorn, streamlit, vaderSentiment, pytest
- PHASE 8 ONLY, OPTIONAL: faster-whisper. It must never become a required dependency
- FORBIDDEN: Spark, Airflow, Dagster, Prefect, dbt, LangChain, ORMs, auth libraries, Kubernetes manifests
- Any new dependency needs a one line justification for why nothing existing covers it

## R3. PII handling (synthetic data, real discipline)

- All data is synthetic and generated by `generate_data.py`. Never ingest anything external
- Treat synthetic names as if they were real: raw names may exist only in Bronze
- Silver hashes `customer_name` into `customer_ref_hash` (SHA-256 with a fixed demo salt, chosen for deterministic reproducible output; production would use keyed HMAC via Key Vault, see Architecture.md). The masking function lives in `silver.py` and has its own test
- Gold tables contain no direct identifiers, enforced by a pandera check on Gold schemas. Note this checks column shape, not content, it does not by itself guarantee no name ever leaks into free text (see R3a)
- The `quarantine` table stores only the offending field and value per failed check, never the full raw row. This is a hard rule, not a preference: it exists specifically to stop an unmasked name reaching a table this project calls trusted
- Logs and console output must never print a raw name. Print hashes or counts

## R3a. AI service demo boundary (hard rule)

- The transcript-masking in `nlp_flags.py` is keyword based, not real NER. It is only proven to work against the 3 official transcripts generated in Phase 1 and covered by their own test
- The AI service, including live in front of an audience, is fed ONLY those 3 pre-written transcripts. Never a name typed live, never a name an audience member suggests on the spot
- If asked to demo an arbitrary or audience-suggested transcript, decline in the moment and offer to show the code path instead. This is not a limitation to hide, it is a reasonable scope boundary to state plainly if asked

## R4. Concurrency (protects the live demo)

- All lake writes go through `pipeline/lake_io.py`. One atomic append per call. Never hold a write open
- The dashboard is read only, always. Reads tolerate a mid-write collision with 2 retries and short backoff, showing last good data instead of an error
- Mandatory rehearsal gate: with the dashboard open and refreshing, post a call to the AI service 5 times in a row. All 5 must succeed
- If Delta locking is flaky, switch Gold to DuckDB and record the decision in Memory.md

## R5. Code style

- Small typed functions with clear names. No classes unless state demands it
- Each pipeline stage runs standalone and via `run_pipeline.py`
- No frameworks, no factories, no premature abstraction. Three similar lines beat one clever indirection
- Comment WHY, not WHAT, especially in `silver.py`, which will be shown on screen

## R6. Error handling

- Bad records are quarantined with a reason, never dropped silently, never crash the run
- Every failure prints one clear line. No bare except blocks
- The AI service returns a clean 4xx or 5xx with a short message, never a stack trace

## R7. Scope discipline

- Build nothing outside PRD.md FR1 to FR8 and the current phase
- Good ideas go into Memory.md under the parking lot, as one line each
- Gold contracts freeze at the end of Phase 4. Changing them after that requires the human to say so explicitly

## R8. Testing and docs

- pytest only. Three test files: silver quality, masking, NLP flags
- A phase is complete only when its Definition of Done in Phases.md passes
- README.md quickstart is three steps maximum: clone, one command, open browser
